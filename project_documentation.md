# توثيق مشروع نظام الإدارة (Admin Dashboard Documentation)

هذا المستند يقدم شرحاً شاملاً لهيكلية المشروع، الجداول، المنطق البرمجي، والصفحات الموجودة في نظام الإدارة.

---

## 🏗️ الهيكلية التقنية (Technical Architecture)
المشروع مبني باستخدام إطار العمل **Laravel 12** مع الاعتماد على:
- **Livewire & Volt**: لمعالجة المنطق البرمجي في الواجهات بشكل حيوي (Reactive UI) دون الحاجة لكتابة الكثير من الـ JavaScript.
- **MySQL**: لقاعدة البيانات.
- **Tailwind/Bootstrap CSS**: (حسب القالب المستخدم) لتنسيق الواجهات.
- **Service Layer**: استخدام خدمات مخصصة مثل `ArabicAmountToWords` لتحويل الأرقام إلى كلمات في الفواتير.

---

## 📂 الصفحات والروابط (Sidebar & Routes)
تم تقسيم النظام إلى عدة أقسام رئيسية تظهر في الشريط الجانبي (Sidebar):

### 1. الرئيسية (Main)
- **لوحة التحكم (Dashboard)**: تعرض إحصائيات عامة (إجمالي المبيعات، المشتريات، عدد العملاء، والمنتجات) مع آخر العمليات.

### 2. إدارة المخزون (Inventory Management)
- **الأصناف (Categories)**: CRUD لإدارة فئات المنتجات.
- **المنتجات (Products)**: إدارة المنتجات (الاسم، SKU، التكلفة، السعر، تنبيه المخزون).
- **المخازن (Warehouses)**: إدارة مستودعات التخزين المختلفة.
- **تحويل المخزون (Stock Transfer)**: منطق لنقل الكميات بين المخازن.

### 3. التوريد والمشتريات (Sourcing & Purchases)
- **الموردين (Suppliers)**: إدارة بيانات الموردين وحساباتهم.
- **المشتريات (Purchases)**: تسجيل فواتير الشراء وتأثيرها على المخزون وحساب المورد.

### 4. المبيعات والعملاء (Sales & Customers)
- **العملاء (Customers)**: إدارة بيانات العملاء (الاسم، الهاتف، الرصيد الافتتاحي).
- **المبيعات (Sales)**: نظام فواتير البيع (اختيار عميل، أصناف، كميات، أسعار، طباعة).

### 5. الموارد البشرية (Human Resources)
- **الموظفين (Employees)**: إدارة بيانات الموظفين وربطهم بعمليات البيع (Salesperson).

### 6. المالية (Finance)
- **السندات (Vouchers)**: إدارة المقبوضات والمدفوعات المالية.

### 7. التقارير (Reports)
- **تقرير المبيعات**: فلترة وتحليل عمليات البيع.
- **تقرير المشتريات**: فلترة وتحليل عمليات الشراء.
- **تقرير الأرباح**: حساب الربح بناءً على (سعر البيع - التكلفة المسجلة وقت البيع).
- **تنبيه المخزون**: عرض المنتجات التي وصلت للحد الأدنى.

### 8. إدارة النظام (System Management)
- **إدارة المستخدمين**: التحكم في مستخدمي لوحة التحكم وصلاحياتهم.
- **سعر الصرف (Exchange Rate)**: ضبط تحديثات أسعار العملات.
- **الإعدادات (Settings)**: بيانات الشركة (الشعار، الاسم، العملة الافتراضية).

## 🗄️ قاعدة البيانات والبيانات (Models & Tables)
تعتمد الجداول على ترابط وثيق لضمان دقة البيانات:

| الموديل (Model) | الدور الأساسي | أهم العلاقات |
| :--- | :--- | :--- |
| `User` | مستخدم النظام | `created_by` في معظم الجداول |
| `Product` | بيانات الصنف | `category`, `stockMovements` |
| `StockMovement` | حركة المخزون (وارد/صادر) | `product`, `warehouse`, `ref_id` |
| `Sale` / `SaleItem` | فواتير البيع وتفاصيلها | `customer`, `employee`, `warehouse` |
| `Purchase` / `PurchaseItem` | فواتير الشراء وتفاصيلها | `supplier`, `warehouse` |
| `Customer` / `Supplier` | أطراف التعامل | `ledgers` (كشف الحساب) |
| `CustomerLedger` | سجل الحركات المالية للعملاء | `customer` |
| `AppSetting` | إعدادات النظام العامة | - |

## 🧠 المنطق البرمجي (Logic & Flow)

### 1. معالجة العمليات (Livewire/Volt)
بدلاً من الكنترولرات التقليدية، يعتمد المشروع على مكونات Volt الموجودة في:
`resources/views/components/admin/`
كل مكون (مثل `sale-management.blade.php`) يحتوي على:
- **المنطق (Logic)**: معالجة الإضافة، الحذف، والحسابات.
- **الواجهة (Template)**: عرض البيانات للمستخدم.

### 2. إدارة المخزون
عند تأكيد فاتورة (بيع أو شراء)، يتم إنشاء سجل في جدول `stock_movements`. رصيد المنتج لا يخزن كقيمة ثابتة دائماً، بل يتم حسابه عبر:
`SUM(qty_in) - SUM(qty_out)` لضمان الدقة التاريخية.

### 3. نظام الطباعة
يوجد نظام طباعة متطور للفواتير (`invoice-print.blade.php`):
- يعتمد على CSS Media Queries للطباعة المتوافقة مع قياس A4.
- يتيح الطباعة بدون خلفية (للطباعة على ورق مروس جاهز) أو تحميل ملف PDF مع الخلفية.
- يستخدم JavaScript لإعادة توزيع المنتجات على صفحات متعددة تلقائياً إذا زاد عددها.

### 4. تحويل المبالغ لرسائل (Services)
استخدام `ArabicAmountToWords` لتحويل إجمالي الفاتورة (مثلاً: 1500) إلى نص (ألف وخمسمائة دينار) لزيادة الموثوقية في السندات.

## 🛠️ ملاحظات للمطورين
- **التحقق (Validation)**: يتم في مكونات Volt مباشرة.
- **الترجمة (Localization)**: المشروع يدعم العربية والإنجليزية عبر ملفات `lang/`.
- **Prefix**: جميع روابط الإدارة تبدأ بـ `/admin` ومحمية بـ `auth` middleware.
